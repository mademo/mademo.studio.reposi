Tu es directeur artistique, UX designer, architecte de l’information et développeur créatif spécialisé dans les sites d’artistes expérimentaux.

Aide-moi à concevoir un site personnel pour **Mademo**, artiste, photographe, designer et autrice.

Je ne veux pas un portfolio classique présentant uniquement des projets terminés. Je veux créer un **atelier numérique vivant** permettant de documenter mes projets en cours, mes recherches, mes photographies, mes textes, mes références, mes essais et mes interrogations.

Le site doit être original, ludique, poétique et légèrement étrange, tout en restant très simple à comprendre et agréable à utiliser.

## Objectif du site

Le site doit me permettre de publier progressivement :

* des projets artistiques en cours ;
* des recherches ;
* des photographies ;
* des dessins et croquis ;
* des textes ;
* des notes rapides ;
* des vidéos ;
* des sons ;
* des références artistiques ;
* des livres et articles ;
* des hypothèses ;
* des expérimentations ;
* des résultats ;
* des échecs ;
* des questions encore ouvertes.

Je veux que le visiteur puisse comprendre comment ma pensée et mes projets évoluent dans le temps.

## Concept central

Imagine le site comme un mélange entre :

* un atelier d’artiste ouvert ;
* un carnet de recherche ;
* une revue d’art indépendante ;
* une bibliothèque personnelle ;
* une constellation d’idées ;
* une exposition numérique ;
* un laboratoire un peu joueur.

Le visiteur ne doit pas seulement voir des œuvres finales. Il doit pouvoir entrer dans les chemins de recherche, les bifurcations, les hésitations et les transformations.

## Organisation du contenu

Le contenu doit être organisé autour de deux éléments principaux.

### Les projets

Exemples :

* La Monade
* L’Œil de la matière
* À trois doigts
* Futur animiste
* Fragments
* Séries photographiques
* Recherches graphiques et éditoriales

Chaque projet peut évoluer et avoir un statut :

* intuition ;
* recherche ;
* expérimentation ;
* production ;
* en pause ;
* terminé ;
* transformé en autre projet.

### Les fragments

Un fragment est une petite unité de recherche pouvant être reliée à un ou plusieurs projets.

Types de fragments :

* photographie ;
* note ;
* texte ;
* question ;
* hypothèse ;
* citation ;
* référence ;
* livre ;
* dessin ;
* vidéo ;
* son ;
* expérience ;
* découverte ;
* difficulté ;
* échec ;
* décision ;
* résultat temporaire.

Chaque fragment doit contenir :

* un numéro ;
* un titre ;
* une date ;
* un type ;
* un contenu ;
* un statut ;
* des mots-clés ;
* les projets auxquels il est relié ;
* les autres fragments associés.

## Navigation

Propose une navigation comprenant :

* Atelier
* Projets
* Fragments
* Recherches
* Photographies
* Textes
* À propos

Le visiteur doit pouvoir explorer le site :

* par projet ;
* par date ;
* par thème ;
* par type de fragment ;
* par mot-clé ;
* par connexion entre les contenus ;
* de manière aléatoire.

Ajouter un bouton intitulé :

**Me perdre quelque part**

Ce bouton ouvre un fragment ou un projet au hasard.

## Page d’accueil

L’accueil doit montrer immédiatement que le site est vivant.

Afficher :

* les projets actifs ;
* ce que je recherche actuellement ;
* les derniers fragments ajoutés ;
* une question du moment ;
* une photographie récente ;
* un projet en pause ;
* une ancienne recherche qui réapparaît ;
* une courte note personnelle.

Phrase d’introduction possible :

« Un espace pour documenter ce qui se cherche avant même de savoir ce qui sera trouvé. »

Créer une composition asymétrique, évolutive et éditoriale.

Ne pas utiliser une grille classique composée de cartes identiques.

## Page d’un projet

Chaque projet doit posséder plusieurs modes de consultation.

### Maintenant

Afficher :

* ce que je cherche actuellement ;
* la dernière avancée ;
* ce qui me pose problème ;
* la prochaine expérience ;
* les questions encore ouvertes.

### Journal

Afficher les étapes du projet dans l’ordre chronologique.

Chaque entrée peut contenir une note, une image, une vidéo, un son ou un document.

Créer des catégories visuelles pour distinguer :

* découverte ;
* hypothèse ;
* test ;
* résultat ;
* difficulté ;
* échec ;
* changement de direction.

### Atlas

Créer une composition visuelle libre rassemblant :

* photographies ;
* croquis ;
* matières ;
* couleurs ;
* références ;
* citations ;
* documents ;
* vidéos ;
* sons.

### Connexions

Montrer comment le projet est relié à d’autres projets, textes, photographies et fragments.

### Synthèse

Créer une version claire et professionnelle du projet comprenant :

* l’intention ;
* le contexte ;
* les médiums ;
* l’état d’avancement ;
* les besoins techniques ;
* les collaborations recherchées ;
* les documents à télécharger.

## Interaction originale

Créer une vue appelée **Constellation**.

Les projets sont représentés comme de grands nœuds.

Les fragments sont représentés comme de petits nœuds.

Les thèmes créent des liens entre eux :

* corps ;
* matière ;
* perception ;
* lumière ;
* vivant ;
* soin ;
* politique ;
* mémoire ;
* technologie ;
* transformation.

La constellation doit rester utilisable et lisible.

Prévoir également une version accessible sous forme de liste.

Ajouter un mode appelé :

**Suivre le fil**

Lorsqu’un visiteur consulte plusieurs contenus, le site construit un parcours montrant les connexions entre eux.

Exemple :

photographie → note sur le corps → expérimentation lumineuse → texte philosophique → projet L’Œil de la matière.

## Dimension drôle et inattendue

Ajouter quelques interactions légères et intelligentes :

* un bouton « Me perdre quelque part » ;
* des messages différents lorsqu’un projet est en pause ;
* un fragment oublié qui réapparaît occasionnellement ;
* un petit indicateur montrant le niveau d’incertitude d’une recherche ;
* une rubrique « Je pensais que ça marcherait » pour les expériences ratées ;
* une rubrique « Je ne sais pas encore » pour les questions ouvertes ;
* un mode aléatoire ;
* des microtextes poétiques dans l’interface.

Exemples de microtextes :

* « Cette idée n’a pas encore décidé ce qu’elle voulait devenir. »
* « Recherche en cours, certitudes momentanément indisponibles. »
* « Ce projet dort, mais il respire encore. »
* « Une erreur intéressante s’est produite. »
* « Aucun résultat définitif n’a été maltraité pendant cette recherche. »

L’humour doit être subtil, artistique et jamais enfantin.

## Direction visuelle

Créer une identité :

* contemporaine ;
* minimaliste ;
* éditoriale ;
* inspirée du graphisme japonais ;
* légèrement brutaliste ;
* organique ;
* expérimentale ;
* drôle par petites touches.

Utiliser :

* un fond blanc cassé ;
* du noir profond ;
* une couleur d’accent vive ou minérale ;
* de grands titres ;
* une typographie sans serif radicale ;
* une typographie serif pour les textes longs ;
* beaucoup d’espace ;
* une grille asymétrique ;
* des numéros ;
* des dates ;
* des annotations ;
* des lignes fines ;
* des formes graphiques simples.

Éviter :

* les cartes arrondies de site commercial ;
* les ombres artificielles ;
* les dégradés technologiques ;
* l’effet application SaaS ;
* les animations trop démonstratives ;
* les interfaces confuses ;
* le style portfolio d’agence générique.

## Photographies

Les photographies doivent pouvoir être présentées :

* dans des séries autonomes ;
* comme fragments de recherche ;
* à l’intérieur des projets ;
* dans une chronologie ;
* par thème ou par lieu.

Respecter les proportions originales des images.

Créer différents rythmes de galerie :

* image pleine largeur ;
* petite photographie isolée ;
* diptyque ;
* mosaïque ;
* succession verticale ;
* mode plein écran.

## Textes

Créer une bibliothèque comprenant :

* textes artistiques ;
* essais ;
* notes de recherche ;
* textes personnels ;
* philosophie ;
* politique ;
* récits de création ;
* journaux de projet.

Chaque texte peut être relié à plusieurs projets et fragments.

La lecture doit rester très confortable, avec une largeur maximale d’environ 680 pixels.

## Interface de publication

Propose aussi une interface privée très simple me permettant d’ajouter rapidement un fragment.

Le formulaire doit contenir :

* type de fragment ;
* titre ;
* contenu ;
* médias ;
* date ;
* projet associé ;
* mots-clés ;
* statut ;
* visibilité publique ou privée.

Je dois pouvoir publier rapidement depuis un ordinateur ou un téléphone.

## Accessibilité

Prévoir :

* navigation au clavier ;
* textes alternatifs pour les images ;
* contrastes lisibles ;
* mode sombre ;
* réglage de la taille du texte ;
* réduction des animations ;
* version liste pour les vues expérimentales ;
* bonne utilisation sur mobile.

Commence par me proposer :
